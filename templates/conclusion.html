Installation

The installation process of Semantic UI was very easy. It simply required a link tag in the head with the Semantic UI url.

Ease of Use

Semantic UI was easy to use, although I personally dislike the long class names. I prefer the abbreviated class names of Bootstrap. Aside from this one minor inconvenience, the framework was easy to use.

Learning Curve

With my previous knowledge of CSS and Bootstrap, Semantic UI was a breeze to learn.

Expectations

I was expecting a bigger change as compared to Bootstrap, but Semantic UI kept feeling like a lower level or older version of Bootstrap.

Installation

Once Python was installed on the server, Flask was very easy to install. Requiring just one "pip install" line in the command line. The difficulty began when hosting the flask product on a production server.

Ease of Use

I have personally used Flask before to create an API, so I found it very easy to use and understand. The ability to reduce repeat HTML/CSS was an amazing feature, but my favourite was Jinja2 implementation. PHP is similiar to Jinja2 but my familiarity with pythonic code made this implementation much easier for me.

Learning Curve

As stated before, I had used Flask before, and as a result there was no learning curve. Viewing it from the angle of someone that had basic-advanced knowledge of Python and HTML, I feel the curve would still be relatively small.

Expectations

I found everything I was hoping to do with the backend was easily doable. And as a result, has not only met but exceeded my expectations.

{%endblock%} {%block bottomtext%}

Would you have done anything differently?

I made the backend a huge portion of the webpages/site. I think if I was to do this project again I would reduce my implementation of it. This is because I feel most pages didn't require it.

Did you regret your choice of framework?

I definetly regreted my choice of frontend framework, as I felt I didn't expose myself to a new framework. I mereily got more practice with Bootstrap. If I was to go back, I would choose React due to the uses of it in web and mobile development.

I didn't regret my choice of Flask as my backend as it exposed me to a different powerful aspect of Flask and Python as a whole.
